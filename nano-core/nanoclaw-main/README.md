## ⠿ What Is MawdBot?

MawdBot is a **unified Web3 agent** that lives on Solana. It combines the security of containerized AI execution with deep on-chain intelligence — real-time token data from **Birdeye**, swaps via **Jupiter**, RPC through **Helius**, and wallet management through **Privy**.

Built on the NanoClaw foundation: one process, a handful of files, agents running in actual Linux containers with filesystem isolation. But now with **full Solana superpowers**.

```
⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀
⣿  🦞 LOBSTER PROTOCOL v1.0                    ⣿
⣿  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━    ⣿
⣿  Status: ONLINE                               ⣿
⣿  Chain:  Solana Mainnet                        ⣿
⣿  RPC:    Helius                                ⣿
⣿  DEX:    Jupiter + Birdeye                     ⣿
⣿  Agent:  Claude SDK (Containerized)            ⣿
⣿  Mode:   Autonomous                            ⣿
⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀⣀
```

## ⠿ Solana Integration

MawdBot connects to the Solana ecosystem through multiple data providers:

### 🔭 Birdeye — Token Intelligence

Real-time Solana token data powered by your `BIRDEYE_API_KEY`:

| Endpoint | What It Does |
|----------|-------------|
| `token/meta-data/single` | Token metadata (name, symbol, logo, socials) |
| `token/meta-data/multiple` | Batch metadata for up to 50 tokens |
| `token/market-data` | Price, liquidity, market cap, FDV, holders |
| `token/trade-data/single` | Volume, unique wallets, buy/sell pressure by timeframe |
| `token/overview` | Full token stats with multi-timeframe price changes |
| `token/list` | Ranked token lists with filtering (volume, mcap, liquidity) |
| `token/txs` | Individual trade history with wallet attribution |
| `pair/overview/single` | DEX pair stats (SOL-USDC, etc.) |
| `price/stats/single` | Price stats with high/low/change by timeframe |
| `token/exit-liquidity` | Real liquidity verification before trades |

**Usage in chat:**
```
@Mawd what's the price and volume of SOL right now?
@Mawd show me the top 10 tokens by 24h volume
@Mawd analyze the trade data for JUP token
@Mawd check liquidity depth for BONK before I swap
```

### ⚡ Jupiter — DEX Aggregation
Swap execution through Jupiter's Ultra endpoint with referral tracking.

### 🔗 Helius — RPC & Parsing
Mainnet RPC, WebSocket feeds, and transaction parsing.

### 🔑 Wallet Stack
Privy for authentication, Phantom for browser wallet, Thirdweb for server wallets.

## ⠿ Architecture

```
                    ⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿
                    ⣿  MAWDBOT ARCHITECTURE  ⣿
                    ⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿

  WhatsApp ──┐
  Telegram ──┤                    ┌──────────────────┐
  Discord  ──┼──► SQLite Queue ──►│  Container Agent  │
  Terminal ──┤                    │  (Claude SDK)     │
  Web API  ──┘                    │                   │
                                  │  ┌─────────────┐  │
                                  │  │ Birdeye API  │  │
                                  │  │ Jupiter Swap │  │
                                  │  │ Helius RPC   │  │
                                  │  │ Wallet Mgmt  │  │
                                  │  └─────────────┘  │
                                  └──────────────────┘
                                          │
                                          ▼
                                  ┌──────────────────┐
                                  │   SOLANA MAINNET  │
                                  │   ⠿⠿⠿⠿⠿⠿⠿⠿⠿   │
                                  └──────────────────┘
```

Single Node.js process. Agents execute in isolated Linux containers with mounted directories. Per-group message queue with concurrency control. IPC via filesystem. **All Solana operations happen inside the container sandbox.**

### Key Files

| File | Purpose |
|------|---------|
| `src/index.ts` | Orchestrator: state, message loop, agent invocation |
| `src/channels/whatsapp.ts` | WhatsApp connection, auth, send/receive |
| `src/ipc.ts` | IPC watcher and task processing |
| `src/router.ts` | Message formatting and outbound routing |
| `src/config.ts` | Trigger pattern, paths, intervals |
| `src/container-runner.ts` | Spawns agent containers with mounts |
| `src/task-scheduler.ts` | Runs scheduled tasks |
| `src/db.ts` | SQLite operations |
| `groups/*/CLAUDE.md` | Per-group memory (isolated) |

```
⣿                                                ⣿
⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿⣿
```

**Secure by isolation.** Agents run in Linux containers. They can only see what's explicitly mounted. Bash access is safe because commands run inside the container.

**AI-native.** No installation wizard — Claude Code guides setup. No monitoring dashboard — ask Claude what's happening. No debugging tools — describe the problem, Claude fixes it.

**Skills over features.** Contributors add [skills](https://code.claude.com/docs/en/skills) like `/add-telegram` that transform your fork. Clean code that does exactly what you need.

**On-chain native.** Birdeye, Jupiter, and Helius are first-class citizens. Token data, swap execution, and RPC are available to every agent in every container.
